# 📝 Dynamic Serverless Blog Setup Guide

The portfolio includes a fully dynamic blog architecture that requires **zero ongoing backend infrastructure**. It runs 24/7 on Vercel seamlessly using **Supabase** (Serverless Postgres) and Next.js Server Actions.

This document walks you through setting up the database and managing your blog posts.

---

## 🏗️ Architecture & Data Flow

1. **Supabase (Serverless Postgres):** Provides the free database tier to persistently store the blogs without needing a Node.js backend. Data travels directly from Next.js Server Actions to Supabase via their API.
2. **Next.js Server Actions:** Securely reads/writes data bypassing client-side requests, inherently hiding your Supabase secure keys and Admin logic from the browser frontend.
3. **Markdown Rendering:** Your content is parsed natively via `react-markdown` with GitHub Flavored Markdown (GFM) support.

---

## 🛠️ Step 1: Set Up Supabase

You need to create a Supabase project to store your blog data.

1. **Create an Account & Project:**
   - Go to [supabase.com](https://supabase.com) and create a free account.
   - Click **"New Project"**, select your organization, name your project (e.g., `payload-of-me-db`), set a database password, and choose a region close to your Vercel deployment.
   - Wait a few minutes for the database to be provisioned.

2. **Retrieve API Keys:**
   - In your Supabase Dashboard, go to **Project Settings** (the gear icon) > **API**.
   - Copy the **Project URL** and the **`anon` `public` key**.

3. **Initialize the Database Schema:**
   - In the Supabase Dashboard, go to the **SQL Editor** located in the left sidebar.
   - Click **"New query"** and paste the following SQL snippet to create the `blogs` table:

   ```sql
   create table public.blogs (
     id bigint generated by default as identity primary key,
     title text not null,
     slug text not null unique,
     excerpt text not null,
     content text not null,
     published boolean default false,
     created_at timestamp with time zone default timezone('utc'::text, now()) not null
   );
   ```

   - Click **"Run"** to execute the query. Your table is now ready!

---

## 🔐 Step 2: Environment Variables

Update your `.env.local` file in the root of your project with the keys you just copied, and set a secure password for your local Admin dashboard.

```env
# Supabase Credentials
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url_here
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key_here

# Admin Dashboard Authentication
ADMIN_PASSWORD=your_super_secret_password_here
```

## ✍️ Step 3: Managing the Blog

### 1. The Admin Dashboard
- **Access the Dashboard:** Go to `http://localhost:3000/admin` (or `https://yourdomain.com/admin` in production).
- **Authenticate:** Enter the `ADMIN_PASSWORD` you defined in your `.env.local`. This sets a secure HTTP-only cookie.
- **Manage Posts:** From the Dashboard, you can see all your draft and published blog posts. You can delete them directly or click "New Post" to open the editor.

### 2. Creating Content
- Navigate to `/admin/new` via the dashboard.
- Craft your article title and short excerpt.
- Write your content using standard **Markdown formatting**.
- **Publishing:** Checking "Publish immediately" will make the article visible on your live site instantly. Otherwise, it is saved securely as a draft.

### 3. Frontend Integration
- **The Homepage (`/`):** The "Latest Writing" section dynamically reads the 3 most recently published posts.
- **The Blog Index (`/blogs`):** A clean listing page displaying all your published articles.
- **The Article (`/blogs/[slug]`):** A dynamically generated route matching the `slug`. It safely decodes your Markdown content into a beautiful reader-focused UI.
